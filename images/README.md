## 📐 圖片準備原則

- **格式**：`.jpg` 為主（照片類）；若改用 `.png`／`.webp`，需同步修改 `index.html` 中對應的檔名
- **檔案大小**：單張建議壓在 **500KB 以內**（可用 [tinypng.com](https://tinypng.com) 或 [squoosh.app](https://squoosh.app) 壓縮），GitHub Pages 免費額度有限、也影響手機載入速度
- **比例**：務必照表裁切，網站是 `object-fit: cover` 滿版裁切，比例不對會被裁掉重點
- **檔名**：全小寫、用 `-` 連字，**不要用中文或空格**（GitHub Pages 對非 ASCII 檔名有時會出問題）

## 🔄 圖片上傳後的生效方式

```
1. 圖片放進 images/ 資料夾
2. git add images/ → git commit → git push
3. 等 1~2 分鐘 GitHub Pages 自動重新部署
4. 瀏覽器強制重新整理（Ctrl+F5）確認
```
